这套主题的导航逻辑仍然以 `content/docs/` 和 `weight` 为核心，但现在已经升级成“章节 + 页面”的两层结构。

## 首页和文档首页

首页模板在 `layouts/_default/list.html`。当前实现会读取 `docs` 分区，并把各章节入口列出来：

```go-html-template
{{ with site.GetPage "section" "docs" }}
  {{ range .Sections.ByWeight }}
    ...
  {{ end }}
{{ end }}
```

这意味着：

- 只要章节属于 `docs` 分区，就能出现在首页和 `/docs/`。
- 不需要手动维护章节卡片列表。
- 调整章节 `_index.md` 的 `weight` 后，入口顺序会一起更新。

## 文档侧栏

侧栏模板在 `layouts/partials/docs-nav.html`。现在它会先列出章节，再列出章节中的页面，并默认展开当前页面所在章节。

当前侧栏有三个实际特性：

- 会高亮当前页面。
- 会展开当前页面所在章节。
- 大屏默认展开，小屏默认收起。
- 展开状态保存在 `localStorage`，滚动位置保存在 `sessionStorage`。

对应脚本在 `assets/js/docs-nav.js`，默认断点是 `1180px`。如果你觉得侧栏太激进或太保守，可以直接改这里。

## 文档页布局

文档详情页模板在 `layouts/docs/single.html`，默认会展示：

- 固定在左上的目录按钮。
- 面包屑。
- 页面标题。
- `date` 和 `lastmod` 对应的发布时间、更新时间。
- 页内目录。
- 正文内容。
- 页底上一篇、下一篇。

模板遇到空值时仍会跳过对应输出，但当前仓库已经把普通文档页纳入 CI 校验，因此 `content/docs/**/*.md` 下的普通页面应当始终提供 `date` 和 `lastmod`。如果缺失，PR 会在校验阶段直接失败。

## 文档分区首页

`/docs/` 本身也是一页，来自 `content/docs/_index.md` 和 `layouts/docs/list.html`。它除了显示正文，还会列出所有章节卡片；各章节自己的 `_index.md` 页面则会列出“本章页面”。

因此这个页面适合放：

- 文档总览说明。
- 阅读建议。
- 面向新用户的入口分类。

## 翻页导航的使用建议

页底翻页来自 `.PrevInSection` 和 `.NextInSection`。如果你希望阅读顺序稳定，最好让章节和文档权重都保持连续，例如：

- `10` 快速开始
- `20` 站点定制
- `30` 内容编写

章节内的页面再继续细分，例如：

- `20` 主题结构总览
- `40` 首页、侧栏与翻页
- `50` 样式与主题定制

这样读者从上往下读时，首页顺序、侧栏顺序和翻页顺序会更接近。
