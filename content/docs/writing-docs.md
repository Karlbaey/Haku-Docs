这套主题最常见的使用方式，就是继续往 `content/docs/` 下面加 Markdown 文件。

## 最小可用示例

新建一个 `content/docs/your-page.md`：

```md
---
title: "你的页面标题"
description: "这段文字会出现在首页卡片和文档目录摘要里。"
date: 2026-05-01T09:00:00+08:00
lastmod: 2026-05-01T09:00:00+08:00
weight: 70
---

这里开始写正文。
```

保存后重新加载页面，Hugo 会自动生成：

- `/docs/your-page/` 页面地址。
- 首页里的文档卡片。
- 文档侧栏中的导航项。

## 这几个 front matter 字段最重要

| 字段 | 用途 |
| --- | --- |
| `title` | 页面标题，同时也会用于导航名称。 |
| `description` | 首页卡片和列表页摘要优先读取这里。 |
| `date` | 文档页顶部会显示“发布于”。 |
| `lastmod` | 文档页顶部会显示“更新于”。 |
| `weight` | 控制排序，数值越小越靠前。 |

如果你不写 `description`，列表会退回到页面摘要；但对文档站来说，直接写清楚通常更稳妥。

## 顺序是怎么控制的

当前主题的几个入口都依赖 Hugo 的排序结果：

- 首页文档目录按 `RegularPagesRecursive.ByWeight` 输出。
- 文档侧栏也按 `RegularPagesRecursive.ByWeight` 输出。
- 页底“上一篇 / 下一篇”使用 section 内页面顺序。

因此最简单的规则就是：给每篇文档都写明确的 `weight`，不要依赖默认顺序。

一个常见分配方式是：

- `10` 到 `30` 放入门页。
- `40` 到 `70` 放日常使用说明。
- `80` 以后放定制、部署或补充材料。

## 页面内容能用哪些 Markdown 能力

当前主题已经对下面这些元素做了样式处理：

- 二级到四级标题。
- 无序列表和有序列表。
- 表格。
- 引用块。
- 行内代码和代码块。
- 分隔线。

## 添加文档时的实践建议

- 文件名保持英文或短横线风格，URL 更稳定。
- 标题写给读者看，文件名写给路径和维护者看。
- `lastmod` 在每次实质修改后更新，读者能更快判断内容是否新鲜。
- 如果一篇内容会长期维护，先把 `description` 写清楚，首页目录会更好扫读。
